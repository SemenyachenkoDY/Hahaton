# Проект: Дашборд-сайт для Хакатона по Аналитике

## Что это такое
Каркас фронтенд-приложения для хакатона по аналитике. Интерфейс должен использовать отличительную эстетику градиента (переход от белого к ярко-оранжевому, как горящая спичка). В нем должны быть предусмотрены выделенные места (контейнеры) для дашбордов и визуализации данных. Также предполагается разделение на `frontend` и `backend` с готовым подключением по API.

## Контекст и мотивация
- **Контекст/Условия:** Разработка для хакатона (критически важны скорость развертывания, структура кода и внешний вид).
- **Целевая аудитория:** Жюри и участники хакатона; проект должен выглядеть современно и премиально.
- **Ограничения:**
  - Сам сайт будет размещен на **GitHub Pages** (что обязывает использовать статическую сборку).
  - Потребуются проработанные "контейнеры" под графики и аналитику на UI.
  - Необходимо задокументировать, какие Docker-контейнеры понадобятся для "серверной" части и как соединяются API.

## Главная ценность
Быстрый и красивый стартовый шаблон. Специалисты по Data Science смогут просто подключать свои скрипты или использовать наши заготовки для API и дашбордов, не отвлекаясь на дизайн.

## Ключевые решения

| Решение | Обоснование | Результат |
|----------|-----------|---------|
| Статический Фронтенд | GH pages поддерживает только статику | Сборка Vite (папка `/frontend`) |
| Разделение аналитики | GH pages не может запускать Python/БД | Описать топологию в `docker-compose.yml` |

## Требования
### Активные
- [ ] Адаптивный каркас сайта с навигацией и брендинговой главной страницей.
- [ ] Огненная/градиентная стилистика (оранжевый и белый).
- [ ] Страница дашбордов с четкими размеченными секциями.
- [ ] Настроена сборка для GitHub Pages (с правильным base path).
- [ ] Скелет `docker-compose.yml` описывающий контейнеры бэкенда (БД, API, и т.д.).
- [ ] Четкое разделение структуры: `frontend/` и `backend/`.
- [ ] Настроены структуры папок для легкого подключения API и других внешних сервисов.
