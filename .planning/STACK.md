# Технологический стек

## Фронтенд (Презентационный слой для Хакатона)
- **Фреймворк:** React 18, функциональные компоненты и хуки.
- **Сборщик:** Vite (позволяет легко собрать статику для GH Pages).
- **Стилизация:** Tailwind CSS / Vanilla CSS (с использованием дизайн-токенов под сочный градиентный стиль).
- **Роутинг:** `react-router-dom` (использование `HashRouter` или грамотная настройка путей для GH Pages).
- **Графики/Дашборды:** Recharts (для создания красивых заглушек аналитических графиков прямо "из коробки").

## Бэкенд и Контейнеры Аналитики (Разработка / Локально)
Так как сайт на GH Pages не может содержать бэкенд, мы описываем нужные инструменты в Docker-контейнерах в корне репозитория (папка `backend/` или корень):
1. **API Сервер:** FastAPI (Python) — легковесный, идеален для хакатонов и аналитики.
2. **База данных:** PostgreSQL — для сохранения аналитических запросов.
3. (Опционально) **Аналитика:** Сервисы Jupyter или Metabase/Superset.
4. **Интерактивные дашборды (DS/ML):** Streamlit + библиотека Altair для быстрой визуализации данных без фронтенда.
5. **Serverless База Данных (Hackathon Mode):** Supabase (Postgres + Auth, прямо из фронтенда) или Vercel Postgres (как URL для FastAPI).
